# 使用统计（IP 入库）部署说明

客户端在**正式版启动**时向后台上报一次；**公网 IP 由服务器从 HTTP 请求写入** MySQL 库 `XBoom`。

> **当前服务器状态（已部署）**  
> - 接口：`https://updates.bcxtech.cn/stats/report.php`  
> - 数据库名：**`XBoom`**（表 `usage_visits`、`usage_users`）  
> - **专用账号**：`xboom`（仅可访问 `XBoom` 库；统计 PHP 使用该账号，不用 root）  
> - 在宝塔 → **数据库** 列表中应能看到 **XBoom** 条目（用户名 xboom）  
> - 更新站 `/stats/` 已单独开启 PHP 8.2

## 表说明

| 表 | 含义 |
|----|------|
| `usage_visits` | 每次启动一条记录（install_id、ip、版本、时间） |
| `usage_users` | 按 install_id 汇总（首次/末次 IP、启动次数） |
| `menu_ip_whitelist` | 受限菜单 IP 白名单（工作台/知识库/任务监控/素材中心） |

**统计「多少人在用」：** 看 `usage_users` 行数（约等于安装台数）。  
**统计「多少次启动」：** 看 `usage_visits` 或 `visit_count` 合计。

## 一、宝塔创建数据库（可选手动）

1. 宝塔 → **数据库** → **添加数据库**  
   - 数据库名：`XBoom`  
   - 用户名/密码：自行设置（记下密码）  
2. **导入** → 选择项目内 `services/usage-stats/schema.sql`  
   - 若库已存在，SQL 会 `CREATE TABLE IF NOT EXISTS`，可重复导入。

## 二、一键部署（推荐）

1. 复制 `scripts/usage-stats.env.example` → `scripts/usage-stats.env`  
2. 填写 `SSH_HOST`、`MYSQL_PASSWORD`（与宝塔 MySQL 一致）  
3. 在项目根目录执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\deploy-usage-stats.ps1
```

脚本会：

- 上传 `report.php`、`config.php` 到 `/www/wwwroot/updates.bcxtech.cn/stats/`（可在 env 里改路径）  
- 远程执行 `schema.sql` 建表  

4. 确保该目录在 Nginx 下**允许执行 PHP**（与更新站同站点即可，子目录 `/stats/`）。

## 三、验证

```powershell
curl -X POST "https://updates.bcxtech.cn/stats/report.php" `
  -H "Content-Type: application/json" `
  -d "{\"install_id\":\"11111111-1111-4111-8111-111111111111\",\"app_version\":\"1.0.2\",\"os_platform\":\"Windows-10\"}"
```

返回 `{"ok":true,"ip":"..."}` 且在 phpMyAdmin 中能看到 `usage_visits` 新行即成功。

## 四、客户端配置

默认已写在 `config.yaml` 合并项中：

```yaml
usage_stats:
  enabled: true
  report_url: https://updates.bcxtech.cn/stats/report.php
  report_token: ""   # 若服务端 config.php 设置了 report_token，此处填相同值
```

关闭统计：`enabled: false`。

仅正式安装包才上报：在 `config.yaml` 的 `usage_stats` 下设置 `release_only: true`（默认 `false`，开发运行也会上报，便于自测）。

**重启后库里没有新记录？**

1. 看启动日志是否有 `[使用统计] 已记录本次启动` 或失败原因。  
2. 若提示「开发运行已跳过」，说明 `release_only: true` 或仍是旧版程序（旧包未含上报代码，需重新打包安装）。  
3. 在浏览器访问 `report.php` 只会 405，必须由程序 **POST** 才会入库。

## 五、查看数据

宝塔 → 数据库 → **XBoom** → 表 `usage_users` / `usage_visits`。

常用 SQL：

```sql
-- 安装台数（去重）
SELECT COUNT(*) FROM usage_users;

-- 今日活跃 IP 数
SELECT COUNT(DISTINCT ip) FROM usage_visits WHERE DATE(created_at) = CURDATE();

-- 最近 20 条访问
SELECT * FROM usage_visits ORDER BY id DESC LIMIT 20;
```

## 六、受限菜单 IP 白名单（推荐 API 模式）

表 `menu_ip_whitelist` 控制 **工作台 / 知识库 / 任务监控 / 素材中心** 是否显示。  
推荐客户端调用服务端 `menu_access_check.php`，由服务器查询 MySQL 后仅返回是否允许，客户端无需保存数据库账号密码。

在 `config.yaml` 配置（推荐）：

```yaml
menu_access:
  enabled: true
  api_url: "https://updates.bcxtech.cn/stats/menu_access_check.php"
  api_token: ""
  timeout_seconds: 8
```

说明：
- `api_token` 对应 `services/usage-stats/config.php` 的 `report_token`，为空则不校验请求头。  
- 客户端仅使用 API 判断白名单；API 不可用时会默认隐藏受限菜单。

若你在客户端通过页面维护白名单（设置 → IP 白名单），仍需配置 MySQL 直连或改造为服务端管理接口。

服务器上添加白名单示例：

```sql
INSERT INTO menu_ip_whitelist (ip, remark, enabled)
VALUES ('1.2.3.4', '同事张三', 1)
ON DUPLICATE KEY UPDATE remark=VALUES(remark), enabled=VALUES(enabled);
```

停用某 IP：`UPDATE menu_ip_whitelist SET enabled=0 WHERE ip='1.2.3.4';`  

客户端内可在 **设置 → IP 白名单** 页面增删改查（仅白名单 IP 可见）；保存后会自动刷新本机权限缓存。若未使用页面，修改后需**重启客户端**才会重新拉取。
