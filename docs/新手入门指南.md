# 小爆来咯 使用说明手册

> 本文介绍各功能模块的作用及相应配置项，便于快速了解系统能做什么、需要准备什么。

---

## 产品概述

**小爆来咯** 是一款 AI 驱动的公众号图文创作工具，主要能力包括：

- 根据话题或热搜自动撰写文章  
- 自动配图、套用排版模板  
- 本地保存文章，并支持发布到微信公众号  
- 支持定时自动写稿、发稿  

使用前需准备：

| 准备项 | 是否必配 | 说明 |
|--------|----------|------|
| 大模型 API | **必配** | 负责写稿、策划、审核等，没有它无法生成文章 |
| 图片生成 API | 建议配置 | 负责封面与插图；不配置可能无图或配图失败 |
| 微信公众号凭证 | 按需配置 | 仅在一键发布、自动发布、定时发稿到公众号时需要 |

下面 **「必配项配置操作指南」** 给出逐项的入口位置与详细步骤；功能说明见后文。

## 必配项配置操作指南

### 如何进入配置页面

1. 启动 **小爆来咯**，进入主界面。  
2. 在 **左侧导航栏最下方**，点击 **「设置」**（齿轮图标）。  
3. 若菜单未展开，点击右侧 **小箭头** 展开 **二级子菜单**。  
4. 子菜单中带 **\*** 的为重要配置：**大模型 API**、**微信公众号**。  
5. 每次改完参数，点击页面右上角 **「保存设置」** 写入本地。

---

### 一、配置大模型 API（必做）

**入口：** 左侧 **设置** → 子菜单 **「大模型 API」**

#### 界面说明（怎么用这一块）

| 位置 | 怎么用 |
|------|--------|
| **顶部工具栏** | **配置厂商** 下拉框选要编辑的厂商；旁有状态提示；按钮：**设为当前使用**、**测试连接**、**+ 自定义**、**删除**（仅自定义可删） |
| **中间表单** | 只显示当前选中厂商：先填 **连接与认证**，再选 **模型**；需要时点 **高级选项** 展开更多模型项 |
| **全局写作** | 页面底部一行，显示**实际用于写稿**的厂商（须点 **设为当前使用** 才会变） |
| **使用说明** | 页面最下方可展开，内有简要操作提示 |

**要点：** 下拉框切换的是「正在编辑哪家」；**设为当前使用** 决定「写作、定时任务用哪家」。两家可以不同——例如先编辑 B 厂商的 Key，再切回 A 并设为当前，写稿仍用 A。

#### 第一步：准备 API Key

任选一家你已开通的大模型服务，在其官网控制台创建 API Key，例如：

- **DeepSeek**：https://platform.deepseek.com  
- **通义千问（阿里云百炼）**：https://bailian.console.aliyun.com  
- **OpenRouter**（聚合多模型）：https://openrouter.ai  
- **硅基流动 SiliconFlow**：https://siliconflow.cn  
- **魔搭 ModelScope**：https://modelscope.cn （支持文生图+大语言模型）
- 其他下拉框中的内置厂商，或 **自定义 OpenAI 兼容** 接口  

记下 Key 字符串（通常以 `sk-` 开头，各家格式略有不同）。

#### 第二步：配置内置厂商（使用方法）

1. 进入 **大模型 API** 页面。  
2. 在工具栏 **「配置厂商」** 下拉框中选择服务商（如 Deepseek、Qwen、OpenRouter 等）。  
3. 在 **连接与认证** 中配置 **API KEY**：  
   - 点击 Key 输入区域，选择 **「点击添加」**，粘贴 Key。  
   - 多个 Key 可 **一行一个**，系统会轮换与重试。  
   - 内置厂商的 **API Base** 一般为只读，一般无需修改。  
4. 在 **模型** 区域选择主模型；列表不全时可点 **测试连接** 或模型旁的刷新，拉取可用模型。  
5. （可选）展开 **高级选项**，按需填写视觉模型、备用模型、模板设计模型、语义精修模型等。  
6. 点击 **「测试连接」**，根据提示确认接口可用。  
7. 点击 **「设为当前使用」**——状态会显示为写作中使用，下拉项带 **✓**。  
8. 点击页面右上角 **「保存设置」**。

#### 第三步：配置自定义 API（OpenAI 兼容）

适用于自建网关、国内中转、或下拉框里没有的兼容接口。

1. 点击工具栏 **「+ 自定义」**，下拉框会出现新的 **自定义 API** 项。  
2. 选中该项，在 **基本信息**（若有）填写 **API 名称**。  
3. 在 **连接与认证** 填写 **API Base**、**API KEY**（一行一个 Key）；自定义项的 Key 框可点 **「显示」** 查看或修改。  
4. 在 **模型** 填写或选择模型名；可先 **测试连接**，成功后再刷新模型列表。  
5. 点击 **「设为当前使用」**，再点右上角 **「保存设置」**。

#### 第四步：切换、管理多家厂商

| 操作 | 做法 |
|------|------|
| 编辑另一家 | 下拉框换厂商即可，已保存的 Key/模型**各自独立**，不会互相覆盖 |
| 换写作用的大模型 | 选中目标厂商 → **设为当前使用** → **保存设置** |
| 删除自定义项 | 选中该自定义项 → 工具栏 **删除** → **保存设置** |
| 恢复某家默认参数 | 页面右上角 **恢复默认**（仅影响当前面板逻辑，按界面提示操作） |

#### 配置完成如何确认

- **全局写作** 或状态提示为你要用的厂商。  
- **测试连接** 成功。  
- 到 **内容生成** 试生成一篇，能正常出字即表示可用。

### 二、配置图片生成 API（建议）

**入口：** 左侧 **设置** → 子菜单 **「图片生成 API」**

#### 可选方案说明

| 方案 | 是否需要 Key | 说明 |
|------|----------------|------|
| **Picsum** | 否 | 免费随机图，适合先跑通流程，画质与主题匹配度一般 |
| **阿里百炼（ali）** | 是 | 通义万相等，需阿里云 DashScope Key |
| **魔搭 ModelScope** | 是 | 需魔搭平台 Token，支持多种文生图模型 |
| **ComfyUI** | 视情况 | 本机已安装 ComfyUI 并开启 `--listen`，填本地地址 |
| **自定义图片 API** | 是 | 兼容 OpenAI 图像接口的第三方服务 |

#### 配置步骤（以阿里百炼为例）

1. 在阿里云百炼 / DashScope 控制台创建 **API Key**。  
2. 进入 **图片生成 API** 页面，找到 **阿里百炼** 对应卡片。  
3. 填入 **API KEY**，在 **模型** 中选择如 `wanx2.0-t2i-turbo`（以界面列表为准）。  
4. 点击 **「测试」**（若有），确认能生成测试图。  
5. 点击 **「设为当前」**。  
6. 页面顶部如有 **「图片生成策略」**（超时、配图数量、是否允许占位图等），可按需调整。  
7. 点击 **「保存设置」**。

#### 使用自定义图片 API

1. 点击 **「添加自定义图片 API」**。  
2. 填写名称、Base URL、Key、模型名（须为服务商支持的文生图模型）。  
3. **测试连接** → **设为当前** → **保存设置**。

#### 配置完成如何确认

在内容生成里生成一篇带配图的文章；正文中有封面/插图即表示图片 API 工作正常。

### 三、配置微信公众号（发布时需要）

**入口：** 左侧 **设置** → 子菜单 **「微信公众号」**（带 \*）

#### 第一步：在微信公众平台获取凭证

1. 浏览器登录 [微信公众平台](https://mp.weixin.qq.com/)（需已认证的服务号或订阅号，且开通开发能力）。  
2. 进入 **设置与开发** → **基本配置**。  
3. 记录 **开发者 ID（AppID）**。  
4. 点击 **AppSecret** 旁的 **重置** 或 **查看**，管理员扫码后 **复制 AppSecret**（**只显示一次**，请立即保存到本地记事本）。  

#### 第二步：配置 IP 白名单（必做，否则接口报错）

1. 在小爆来咯 **微信公众号** 配置页顶部，查看 **「服务器出口 IP」**（软件会自动检测本机访问微信时的公网 IP）。  
2. 点击 **「复制」** 复制该 IP。  
3. 回到微信公众平台 **基本配置** → **IP 白名单** → **修改**，粘贴并保存该 IP。  
4. 若你更换网络（如家里 / 公司切换），出口 IP 可能变化，需重新复制并更新白名单；配置页可点 **「刷新」** 重新获取 IP。

> 页面内可展开 **「如何获取 AppSecret & 配置白名单？」** 查看图文式步骤。

#### 第三步：在软件中填写凭证

1. 在凭证卡片中填写：**AppID**、**AppSecret**、**作者名**（公众号显示的作者，必填）。  
2. 按需勾选发布相关选项（见下表）。  
3. 点击该卡片上的 **「测试连接」**，显示成功后再 **「保存设置」**。  
4. 若有多个公众号，可点 **「+ 添加凭证」** 增加多套，发布或定时任务时选择对应账号。

| 选项 | 含义 |
|------|------|
| 仅草稿 | 只上传到公众号草稿箱，不直接群发 |
| 调用群发接口 | 是否走群发相关能力（与下方选项联动） |
| 群发全部 | 是否发给全部粉丝（需谨慎） |
| 标签 ID | 按标签群发时填写对应 ID |

**建议新手：** 先勾选 **「仅草稿」**，在公众号后台人工预览确认后再正式发布。

#### 配置完成如何确认

- **测试连接** 显示成功。  
- 在 **文章库** 中对某篇文章执行发布，能在公众号后台看到草稿或已发布内容。

```
大模型 API（必配）→ 设为当前使用 → 保存设置
    ↓

    ↓

```

## 功能模块说明

### 工作台

展示内容生产的整体数据，例如活跃话题数、今日生成文章数、质量相关指标，以及话题流转的可视化视图。

**配置：** 无需单独配置，随系统运行自动汇总数据。

---

### 内容生成（核心模块详解）

手动发起内容创作的核心模块。可输入指定话题，也可留空由系统按规则自动选取热点。生成过程包括多轮深度优化环节，完成后文章进入文章库。

#### 界面布局说明

```
┌─────────────────────────────────────────────────────┐
│  内容生成                                             │
│  ┌─────────────────────────┬───────────────────────┐ │
│  │  主创作区                │  右侧：生成选项         │ │
│  │  ┌─────────────────┐   │                       │ │
│  │  │ 话题/关键词      │   │  • 生成数量 (1-20)     │ │
│  │  └─────────────────┘   │  • 完成后 (存草稿/发布)  │ │
│  │                         │  • ☑ 极速模式          │ │
│  │  [借鉴模式][运行日志]    │  • ☑ 过滤已写          │ │
│  │  [实时预览]             │  • ☑ 自动换模板        │ │
│  │                         │                       │ │
│  │  [开始生成]              │  [⚡ 开始生成]          │ │
│  ├─────────────────────────┤                       │ │
│  │  [实时写作] [运行日志]   │                       │ │
│  └─────────────────────────┴───────────────────────┘ │

│  │  内容质量检测面板（生成完成后自动弹出）            │ │

```

#### 话题/关键词输入区

| 元素 | 作用 |
|------|------|
| **话题输入框** | 输入你想写的主题，如"春季护肤避坑指南"；留空则自动获取热搜话题 |
| **生成进度条** | 显示当前写作阶段（初始化→抓取解析→策划大纲→撰写→终审→优化→配图排版→完成）|

#### 工具栏按钮

| 按钮 | 作用 |
|------|------|
| **借鉴模式** | 展开参考配置面板，可选择模板分类/名称、借鉴比例(10%-75%)、输入参考链接、选择热点文章借鉴 |
| **运行日志** | 展开/收起日志面板，实时显示各Agent的工作状态和输出信息 |
| **实时预览** | 切换到预览模式，以阅读方式实时查看正在生成的文章内容 |

#### 借鉴模式面板（点击"借鉴模式"展开）

| 选项 | 作用 |
|------|------|
| **模板分类** | 选择排版模板的分类（如情感心理、科技数码等），不选则随机 |
| **模板名称** | 选择具体模板，不选则随机 |
| **借鉴比例** | 控制参考内容的融入程度：10%(轻微)、20%(轻度)、30%(中等)、50%(较强)、75%(强烈) |
| **参考链接** | 输入外部文章URL作为风格参考，多个链接用 `|` 分隔 |
| **热点文章借鉴** | 从已抓取的热点中选择一篇文章进行借鉴分析 |

#### 右侧生成选项

| 选项 | 作用 |
|------|------|
| **生成数量** | 一次生成的文章篇数（1-20），默认为1 |
| **完成后** | 选择生成后的操作：不操作 / 存草稿 / 发布到公众号 |
| **极速模式** | ⚡ 开启后跳过RSC递归自检、Reflexion打磨、维度化创意变换等深度优化环节，大幅缩短生成时间（约3-5分钟），但文章质量会有所降低 |
| **过滤已写** | 过滤最近30天内已处理过的话题，避免重复选题 |
| **自动换模板** | 生成完成后自动切换不同模板重新排版（保留配图）；极速模式下不可用 |
| **开始生成** | 启动写作流程 |

#### 输出区域

| 标签页 | 作用 |
|--------|------|
| **实时写作** | 以阅读模式展示正在生成的文章内容，支持滚动到底部、清空、侧栏预览 |
| **运行日志** | 显示详细的Agent工作日志，支持复制、清空、导出 |

#### 内容质量检测面板

文章生成完成后，底部会自动弹出**内容质量检测面板**，提供以下功能：

| 功能 | 说明 |
|------|------|
| **综合评分** | 圆环图显示文章整体质量分数（0-100分）|
| **AI检测概率** | 文章被识别为AI生成的概率（越低越好，<30%为安全区间）|
| **原创性** | 内容原创程度评分（越高越好）|
| **详细指标** | 多维度评分网格：开头吸引力、论证深度、信息密度、叙事节奏、AI痕迹、阅读连贯性、排版跳跃感、情绪价值、读者收获 |
| **优化建议** | 系统给出的具体改进建议，可勾选后执行一键改写 |
| **自动优化** | 一键启动多轮自动优化（最多5轮），逐步提升综合评分、降低AI概率 |
| **对比结果** | 查看优化前后的各项指标对比 |
| **应用优化结果** | 将优化后的内容应用到文章中 |
| **清理场景描述** | 移除正文中可能残留的场景描述与生图提示词泄露 |

**使用建议：**
- 综合评分 ≥ 75 分且 AI 检测概率 < 30% 时，文章质量良好，可直接使用
- 如 AI 检测概率 > 50%，建议点击"自动优化"进行去AI味处理
- 可勾选特定优化建议后点击"一键改写选中"，针对性修复问题

#### 写作流程详解（正常模式）

正常模式下，系统采用多Agent协作的深度写作流程，确保文章质量：

```
Step 1: 深度洞察（信息采集 + 话题拆解）
   ├── 抓取解析：采集热点资讯、提取关键信息
   ├── 话题拆解：将复杂话题分解为可操作的子主题
   └── 维度锚定：确保后续内容围绕原话题展开

Step 2: 策划大纲
   ├── 创意蓝图生成：从受众、场景、情绪、结构等多维度构思
   ├── 维度化创意变换（可选）：根据配置的维度参数变换文风
   └── 大纲构建：形成完整的文章骨架

Step 3: 正文撰写
   ├── 初稿生成：按照大纲撰写完整正文（1200字以上）
   ├── RSC递归自我修正（2轮）：逻辑审核官批判→核心重构员修复
   └── 确保论证递进、信息密度、叙事节奏达标

Step 4: 终审评估
   ├── 主编审稿：从9个维度打分（开头吸引力、论证深度、信息密度等）
   └── 给出精准可执行的修改建议（标注到具体段落位置）

Step 5: Reflexion打磨（2轮）
   ├── 第1轮：根据主编反馈定向修复问题段落
   ├── 第2轮：再次审稿评估，继续优化未达标项
   └── 循环直到评分≥75分或达到最大轮次

Step 6: 抗AI处理
   ├── 语言指纹偏移：调整句式结构，减少AI特征
   ├── 语义模糊化：替换过于精确的表达
   └── 保持可读性的同时降低AI检测率

Step 7: 配图排版
   ├── 视觉逆向工程：从正文中提取场景描述
   ├── 图片生成：调用图片API生成配图（封面+插图）
   └── 模板套用：应用HTML排版模板，输出最终成稿
```

**预计耗时：** 正常模式约8-12分钟（取决于文章长度和模型响应速度）

#### 极速模式 vs 正常模式对比

| 对比项 | 极速模式 | 正常模式 |
|--------|----------|----------|
| **生成时间** | 3-5分钟 | 8-12分钟 |
| **RSC递归自检** | ❌ 跳过 | ✅ 2轮深度修正 |
| **Reflexion打磨** | ❌ 跳过 | ✅ 2轮定向优化 |
| **维度化创意变换** | ❌ 跳过 | ✅ 按配置执行 |
| **主编终审** | ✅ 简化版 | ✅ 9维度深度审稿 |
| **抗AI处理** | ✅ 轻量级 | ✅ 完整流程 |
| **适用场景** | 快速出稿、试话题、批量生产 | 追求高质量、正式发布 |

**建议：**
- 日常快速出稿或批量生产 → 使用极速模式
- 重要文章、正式发布 → 使用正常模式

### 文章库

管理已生成文章：浏览、搜索、预览、编辑标题与正文，保存到本地，以及发布到已配置的公众号。支持 AI 美化、保留配图等二次处理。

**说明：** 内容生成与自动化任务产出的文章均汇总于此，是写稿之后的统一工作台。

---

### 知识库

查看本机数据占用情况，包括文章文件、图片资源、数据库等所占空间，便于备份与清理。

**配置：** 无需配置，用于运维与空间管理。

---

### 模板库

管理公众号文章的 HTML 排版模板，决定成稿的视觉版式（配色、区块、字体风格等）。

**配置关联：** 在基础设置中可启用「使用模板」，并指定模板分类或具体模板；不启用时由系统按默认方式输出排版。

---

### 资讯采集

面向国内各选题来源的热点采集与解析，用于在留空话题时参与自动选题，或供创作流程引用。

**配置关联：** 与「选题来源」设置配合使用，通过启用平台及权重控制选题来源。

---

### 选题中心

从大量 RSS、资讯类信息源聚合新闻条目，支持按分类浏览、过滤已写过的话题、手动或定时刷新。

**适用场景：** 浏览行业动态、寻找写作选题，可作为内容生成选题的补充来源。

**说明：** 与资讯采集侧重不同——采集偏国内热搜榜，聚合偏多源资讯 feed。

---

### MCP 服务

管理 MCP（Model Context Protocol）扩展服务，为 AI 工作流接入外部工具或数据源。

**适用场景：** 需要对接额外数据服务、工具链的进阶用户；初次使用可暂不配置。

---

### 自动化任务

按设定周期在指定时间自动执行与内容生成相同的流程，成稿写入文章库，并可选择自动发布到公众号。

**使用前提：**

- 应用需保持运行，系统才会在计划时间执行任务  
- 已配置可用的大模型 API  
- 若需自动发布，需已配置并验证通过的微信公众号凭证  
- 当前自动发布能力以 **微信公众号** 为主，其他平台需自行复制内容发布  

**任务可配置项通常包括：** 固定话题或自动选题、执行周期（每日/每周/单次等）、是否自动发布、使用的公众号账号等。

---

### 任务监控

展示多 Agent 蜂群模式的运行状态、节点拓扑与任务调度信息，面向并发实验与系统观测。

**重要说明：** 默认应使用 **串行任务模式**（一次只执行一个写作任务），以保证输出完整、可用的公众号文章。关闭串行、启用蜂群并发属于实验特性，**不适合**日常稳定出稿。日常创作无需关注本模块。

---

### 素材中心

保存文章生成过程中的移动端预览截图及视觉审计记录，便于在发布前检查成稿在手机上的大致效果。

**配置：** 无需配置，有文章生成后会自动积累记录。

### 内容生成

控制「维度化创意」引擎：从受众、场景、情绪、结构等多个维度变换文风。

| 配置项 | 说明 |
|--------|------|
| 启用维度化创意 | 总开关 |
| 创意强度 | 变换幅度（数值越高越激进） |
| 自动选择维度 | 由 AI 自动挑选维度组合 |
| 最大维度数 | 同时应用的维度上限 |
| 兼容性阈值 | 维度组合允许的差异程度 |
| 保持核心信息 | 变换时尽量保留事实与主旨 |
| 允许实验性组合 | 允许更大胆、非常规的组合 |

**建议：** 初次使用可先关闭，熟悉基础出稿后再逐步开启。

### 大模型 API

配置用于写稿、策划、审核等文本任务的 LLM 服务。

**使用方法概要：**

1. **配置厂商** 下拉框 → 选择要编辑的内置或自定义厂商。  
2. 填写 **连接与认证**（API KEY，一行一个）、选择 **模型**；需要时展开 **高级选项**。  
3. **测试连接** → **设为当前使用** → 右上角 **「保存设置」**。  
4. 写作、定时任务使用 **全局写作** 里显示的那家厂商。

| 配置项 | 说明 |
|--------|------|
| 配置厂商 | 切换正在编辑的厂商；各厂商 Key、模型独立保存 |
| 设为当前使用 | 指定内容生成、自动化任务实际调用的大模型 |
| 测试连接 | 检测 Key 与接口；可刷新模型列表 |
| 连接与认证 | API KEY、API Base（自定义须填 Base） |
| 模型 / 高级选项 | 主模型及视觉、备用、模板设计、语义精修等扩展模型 |
| + 自定义 | 新增 OpenAI 兼容接口 |
| 保存设置 | 将当前所有厂商配置写入本地 |

**必配项：** 无有效大模型 API 时无法生成文章。

## 典型使用方式

### 手动创作一篇（推荐流程）

1. **设置 → 大模型 API**：下拉选厂商 → 填 Key → **测试连接** → **设为当前使用** → **保存设置**。  
2. （建议）**图片生成 API**：选服务商、填 Key → **设为当前** → **保存设置**。  
3. **内容生成**：
   - 输入话题或留空让系统自动选题
   - 关闭**极速模式**（追求高质量）
   - 点击 **开始生成**，等待8-12分钟
4. **查看质量检测结果**：
   - 检查综合评分和AI检测概率
   - 如需优化，点击 **自动优化** 或勾选建议后 **一键改写选中**
5. **文章库**：检查正文与配图 → 编辑 → 按需发布到公众号（需先配好微信公众号）。

### 快速批量出稿

1. 同上完成API配置。
2. **内容生成**：
   - 输入话题或留空
   - **开启极速模式**（缩短生成时间）
   - 设置 **生成数量** 为多篇（如5篇）
   - 点击 **开始生成**，等待3-5分钟/篇
3. **文章库**：批量检查并发布。

### 定时自动创作

1. 按上完成大模型 API；若要自动发稿，再配好 **微信公众号** 并 **测试连接**。  
2. **自动化任务**：新建任务，设置执行时间、话题规则、是否自动发布及使用的公众号。  
3. 保持应用运行；到计划时间后，在 **文章库** 或任务相关界面查看成稿。

## 常见问题

### 生成相关问题

**无法生成文章**  
检查大模型 API 是否已保存、Key 是否有效、账户是否有余额，网络是否可访问对应服务。

**生成时间很长怎么办？**  
- 正常模式需要8-12分钟，这是为了保证文章质量的正常现象
- 如需加快速度，可在生成选项中开启 **极速模式**（3-5分钟/篇）
- 极速模式会跳过RSC自检、Reflexion打磨等深度优化环节

**文章质量不满意怎么办？**  
- 确保使用的是 **正常模式**（非极速模式）
- 生成完成后查看 **内容质量检测面板** 的评分和建议
- 点击 **自动优化** 进行多轮优化
- 或在 **文章库** 中对文章进行二次编辑

**没有配图**  
检查图片生成 API 是否配置正确。

**图片上有文字怎么办？**  
这是AI生图的常见问题。系统已通过负向提示词优化来减少此情况，但无法完全避免。如频繁出现，可：
- 更换图片生成模型（如从阿里百炼切换到魔搭）
- 在图片生成 API 设置中调整负向提示词

**无法发布到公众号**  
检查 AppID、AppSecret、IP 白名单及公众号接口权限。

### 定时任务相关

**定时任务没有执行**  
确认应用仍在运行，任务处于启用状态，且到达设定时间。

**话题总是重复**  
可开启「严格新鲜度」，或在选题中心中启用「过滤已写话题」。

### 系统相关

**蜂群模式是否要用**  
日常不必使用；保持串行任务模式即可稳定出稿。

**配置修改未生效**  
确认已点击 **「保存设置」**；少数界面类设置需重启应用。

**软件如何更新**  
启动时会自动检查新版本，如有更新会提示下载安装。也可手动下载最新版安装包覆盖安装。

**AI检测率准不准**  
AI检测率是基于统计模型的估算值，仅供参考。实际效果取决于：
- 目标平台的检测算法
- 文章的具体内容和风格
- 优化处理的力度

一般来说，AI检测概率 < 30% 时被识别的风险较低。

## 进阶技巧

### 1. 提升文章质量的方法

- **始终使用正常模式**：除非赶时间，否则不要开极速模式
- **善用借鉴模式**：提供高质量参考链接，系统会学习其写作风格
- **合理设置维度化创意**：开启后能让文章更有个性，但不宜设置过高强度
- **生成后务必检查质量面板**：根据优化建议针对性改进

### 2. 降低AI检测率的技巧

- 生成后使用 **自动优化** 功能（最多5轮）
- 在质量面板中勾选 **AI痕迹** 相关建议并一键改写
- 避免使用过于规整的排比句式
- 增加个人化的表达和口语化的过渡

### 3. 批量生产的效率优化

- 开启 **极速模式** + 设置 **生成数量** 为多篇
- 使用 **过滤已写** 避免重复选题
- 配合 **自动化任务** 实现定时自动出稿

### 4. 图片优化建议

- 优先使用 **魔搭 ModelScope** 或 **阿里百炼**，图片质量较好
- 如遇到图片带文字问题，尝试更换模型或在ComfyUI中调整工作流
- **文章配图数量** 建议3-5张（含封面），过多会影响加载速度
